#### Are you dealing with these backup challenges?
* Physical tape and infrastructure complexity
* Expensive offsite tape storage
* Shrinking Capex budget
  
#### AWS Storage Gateway
Provides on-premises access to virtually unlimited cloud storage.  
Configuration: VMware ESXi, Microsoft Hyper-V, Amazon EC2, HW appliance  
Integrated with IAM, KMS, CloudTrail, CloudWatch  
  
#### Storage Gateway Family
* **File** - Store and access objects in Amazon S3 from SMB / NFS file-based applications with local caching
* **Tape** - Drop-in replacement for physical tape infrastructure backed by cloud storage with local caching (iSCSI VTL)
* **Volume** - On-premises iSCSI block storage backed by cloud with local caching, init. with Amazon Backup and Amazon EBS
You can use all gateway types to backup on-premises workloads to cloud.   
  
#### Tape Gateway
Use cases:  
* Backup to cloud
* Archive to cloud
  
Benefits:  
* Replaces physical tapes - drop-in replacement to your physical tape infra; lower TCO than tape backups; pay-as-you-go pricing (compared to traditional capex-based models)
* Allows easy management - no changes to backup processes; only manage tapes and gateway; gateway managed and updated by AWS; multiple deployment options
* Provides security and compliance - read-only tapes upon archive; data encrypted on wire; server-side or KMS encryption; HIPAA eligible; PCI, SOC 1,2,3 and ISO 9001, 27001, 27017, 27018 compliant
* Optimizes performance - local caching for faster backup and recovery times; compress data prior to upload; easy and online retrieval
